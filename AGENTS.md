# Repository Guide

## Repository Purpose

This is the GitHub profile README repository for `brightcolin/brightcolin`. The public profile is rendered from `README.md`.

## README Structure

The README is organized into these sections:

1. **Header** — name, university affiliation, badge links, and a short introduction.
2. **What I'm Building** — a two-column HTML table showcasing Smart Calendar Assistant and AI Finance Analyzer.
3. **Team Project** — Colin's role and contributions to ThuStory.
4. **Highlights** — awards and competition results.
5. **Tools I Use** — a compact grid of technology badges.

## Conventions

- Keep the profile concise, honest, and written in English.
- Preserve the existing Markdown and HTML structure unless a requested change requires restructuring it.
- Project entries follow this pattern: linked title, short description, and inline technology tags; add badges only when they provide useful status or release information.
- Use `shields.io` with `style=flat-square` for project and technology badges. The two header badges intentionally use the default style.
- Keep award names and years consistent with the linked project repositories and official results.
- Do not add decorative GitHub statistics, animations, or large badge collections unless explicitly requested.

## Verification

- There is no build step or automated test suite for this repository.
- After editing Markdown or HTML, run `git diff --check` and review the rendered structure for broken links, malformed tables, and inconsistent wording.
- Do not commit or push changes unless explicitly requested.
